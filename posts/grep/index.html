The GREP Command

searching made easy...

The grep utilities are a family that includes grep, egrep, and fgrep for searching files. For most use cases, fgrep is sufficient due to speed and only looking into strings and words. However, typing grep is easy. Grep is an essential Linux and Unix command. It is used to search text and strings in a given file. In other words, grep command searches the given file for lines containing a match to the given strings or words. It is one of the most useful commands on Linux and Unix-like system for developers and sysadmins.

Following are some examples of using the grep command in different ways.

Basic Usage

Search any line that contains the word in filename:

1

grep 'word' filename

Perform a case-insensitive search for the word ‘bar’ in Linux and Unix:

1

grep -i 'bar' file1

Look for all files in the current directory and in all of its subdirectories in Linux for the word ‘httpd’:

1

grep -R 'httpd' .

Search and display the total number of times that the string ‘nixcraft’ appears in a file named frontpage.md:

1

grep -c 'nixcraft' frontpage.md

Searching through a file

Search the /etc/passswd file for a user named ‘boo’, enter:

1

grep 'boo' /etc/passwd

We can use fgrep/grep to find all the lines of a file that contain a particular word. For example, to list all the lines of a file named address.txt in the current directory that contain the word “California”, run:

1

fgrep California address.txt

The above command also returns lines where “California” is part of other words, such as “Californication” or “Californian”. Hence pass the -w option with the grep/fgrep command to get only lines where “California” is included as a whole word:

1

fgrep -w California address.txt

You can force grep to ignore word case i.e match boo, Boo, BOO and all other combination with the -i option:

1

grep -i "boo" /etc/passwd

We can also use grep to search 2 different words, for example use egrep command to search for word1 or word2 as follows:

1

egrep -w 'word1|word2' /path/to/file

Searching through a folder (recursively)

You can search recursively i.e. read all files under each directory for a string “192.168.10.15”:

1

grep -r "192.168.10.15" /etc/

You will see results for the above command on a separate line preceded by the name of the file (such as /etc/ppp/options) in which it was found. The inclusion of the file names in the output data can be suppressed by using the -h option as follows:

1

grep -hR "192.168.10.15" /etc/

Line number or count for matching word

The grep can report the number of times that the pattern has been matched for each file using -c (count) option:

1

grep -c 'word' /path/to/file

Pass the -n option to precede each line of output with the number of the line in the text file from which it was obtained:

1

grep -n 'root' /etc/passwd

Lines before/after the match

Want to see the lines before your matches? Try passing the -B to the grep:

1

grep -B NUM "word" file

1
2

#display 3 lines before a match is found
grep -B 3 "foo" file1

SImilarly, for lines after a match is found, try using -A:

1
2
3

grep -A NUM "string" /path/to/file
# Display 4 lines after dropped word matched in firewall log file #
grep -A 4 "dropped" /var/log/ufw.log

Sometimes we combine different options to get some meaningful outputs, such as:

1

grep -C 4 -B 5 -A 6 --color 'error-code' /var/log/httpd/access_log

Force grep invert match

You can use -v option to print inverts the match; that is, it matches only those lines that do not contain the given word. For example print all line that do not contain the word bar:

1

grep -v bar /path/to/file

Get names of matching files

Use the -l option to list file name whose contents mention main:

1

grep -Rl 'main' /path/to/project/dir/

Colors option

Finally, we can force grep to display output in colors, enter:

1

grep --color vivek /etc/passwd

Saving grep output to a file

Let us say you want to search for keyword acl and save that output a file named my-acl.txt, and then you need to run:

1

grep acl squid.conf > my-acl.txt

We can find tune searching and limit the search to words starting with keyword named acl:

1

grep -w '^acl' squid.conf > my-acl.txt

Search special characters

Want to search [ or ’ or ^ special characters using grep? You can ask grep to treat your input as fixed string using -F option or or fgrep command. For example:

1
2
3
4

# match ‘[xyz]’ in the filename
fgrep '[xyz]' filename
# or
grep -F '[xyz]' filename

We can also use the ** as escape character. To match a character that is special to egrep or grep -E, a a backslash () in front of the character. It is usually simpler to use grep -F when you don’t need special pattern matching. For instance, try to match ‘foo’ by putting ** before ’:

1

grep -E '\'foo\'' input

In short, use the fgrep or grep -F when you do not need to use pattern matching using regex.
